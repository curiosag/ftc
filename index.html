Get a quick start with Google fusion tables sql. You need some fusion tables you own (and/or some external table IDs), a google developers project, an activated fusion tables api and an OAuth2 client ID for the project. (the OAuth2 client ID must be for application type "other") Download the jar (zipped/gzipped), extract, start ftc.bat or ftc.sh (make executable first)

Coverage

SELECT, CREATE VIEW, INSERT, UPDATE, DELETE, DESCRIBE and SHOW TABLES, well ... everything.

For tables owned by yourself you can use table names instead of table IDs. Make sure, there are no naming collissions. This also was the motivation to start the project in the first place. I have a limited capacity to memorize random sequences of 40 characters, but I'm also working on this issue with Derren Brown. The app was the quick fix, though.

Extended commands

CommandSyntax
Rename tableALTER TABLE [table name or ID] RENAME TO [new name]
Delete tableDROP TABLE [table name or ID]
Copy tableCREATE TABLE [table name] AS SELECT * FROM [source table name or ID]

Shortcuts

F5Execute query
Ctrl+F5Cancel query execution
Alt+F5View preprocessed query
Ctrl+FFocus editor text field
Ctrl+spaceTrigger code completion. Switch between proposed sql snippets and table or column names
Alt+leftBrowse command history backwards
Alt+rightBrowse command history forward
Alt+MMemorize editor text ... means, append it to the command history without executing the query
Ctrl+TShow tables
Ctrl+EExport to csv file
Ctrl+SSave editor text ... a bit of an overkill, as you can only use a single statement per execution anyway. On the other hand: who knows, maybe somebody uses it to write a novel or whatever
Ctrl+OOpen file

Limitations

  • Column ids can be used in queries, but they are not covered by semantic highlighting, so you get a blue wiggle line for them
  • If a table has an alias, the alias must always be preceded by the keyword "AS"
  • Handling of huge amounts of data isn't the goal of the implementation. On the other hand, it may take you as far as the computer's memory allows it to.
